Links & Images

In HTML, you can link to other pages or resources using the <a> tag, and you can add images to your webpage with the <img> tag.

Links

To create a link to another webpage or resource, use the <a> tag. The href attribute specifies the destination URL.

Example:

<a> href="https://www.example.com">Click Here</a>

This creates a clickable link that takes the user to https://www.example.com. You can also link to different sections of the same page using anchor links. For example:

<a>href="#Introduction">Go to Introduction</a>

Images

To add an image to a webpage, use the <img> tag. The src attribute specifies the path to the image file, and the alt attribute provides alternative text if the image is unavailable.

Example:

<img> src="image.jpg" alt="Description of Image"</img>

This adds an image called image.jpg and provides a description in case the image is not displayed. You can also set the image's width and height using the width and height attributes:

<img>img src="image.jpg" alt="Description of Image" width="500" height="300"</img>

Tables

To create tables in HTML, you use a set of tags to define the structure of the table. Here's a simple breakdown of the key elements:

Key Tags for Tables:

  • <table>: Defines the table.
  • <tr>: Defines a table row.
  • <th>: Defines a table header cell (bold and centered by default).
  • <td>: Defines a table data (cell).
  • <thead>: Groups the header content of a table.
  • <tbody>: Groups the body content of a table.
  • <tfoot>: Groups the footer content of a table.

Simple Table Example:


  <table>
    <thead>
      <tr>
        <th>Name</th>
        <th>Age</th>
        <th>City</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td>John</td>
        <td>25</td>
        <td>New York</td>
      </tr>
      <tr>
        <td>Alice</td>
        <td>30</td>
        <td>Los Angeles</td>
      </tr>
    </tbody>
    <tfoot>
      <tr>
        <td colspan="3">Table footer</td>
      </tr>
    </tfoot>
  </table>

Breakdown:

  • The <thead> contains the header row (<tr>) and header cells (<th>).
  • The <tbody> holds the data rows (<tr>) and data cells (<td>).
  • The <tfoot> can be used for footer content, typically for summaries or other additional information.

Styling Tips:

  • You can use CSS to style tables, such as adding borders, padding, and controlling the width of columns.
  • If you want the table to be more readable, consider adding border-collapse in CSS to remove extra space between table cells.

Forms are essential for collecting data from users and submitting it to a server for processing. This section covers the different types of forms, their elements, and how to handle form submission.

1. Basic Form Structure

A basic form is created using the

tag. The <form> element wraps the entire form and allows you to define its action (where the data is sent) and the method (how the data is sent).

6. Storm Accessibility

To ensure your forms are accessible to all users, including those using screen readers, make sure to:

  • Use proper form labels.
  • Group related inputs together.
  • Provide clear instructions when needed.

Example of Accessible Form:

                  <input type="email" id="email" name="email" aria-required="true" required>

  • aria-required="true" tells screen readers that this field is required.

Semantic HTML Elements

Semantics in HTML refers to the use of HTML elements that have clear, descriptive meaning about the content they contain. Using semantic elements improves accessibility, search engine optimization (SEO), and code maintainability

Why use Semantics?

  • Helps screen readers and assistive tech
  • Boosts search engine visibility
  • Makes code easier to maintain

Common Semantic Elements

  • <header>: top section of a page or section, often includes logo and nav links.
  • <footer>: Bottom section with contact, copyright, or related links.
  • <nav>: Block of site navigation links.
  • <main>: Main content unique to the page.
  • <section>: Groups related content, usually with a heading.
  • <article>: Independent, self-contained content like blog posts.
  • <aside>: Tangential content like sidebars or related links.
  • <figure>/<figcaption>: Media with a caption.

Non-Semantic (Use Sparingly)

  • <div>: Generic block
  • <span>: Generic inline

    • Use only when no semantic alternative fits
